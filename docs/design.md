# 設計書: Claude Code 利用説明アプリ

## 1. 概要

### 目的
社内エンジニア（初級者含む）がClaude Codeの設定ファイル構成を視覚的に理解できるインタラクティブWebアプリ。

### コンセプト
- ディレクトリツリーをクリック → サイドパネルに「なぜ必要か」「何をするファイルか」「書き方」を表示
- ウィザード形式で設定ファイルを対話的に生成できる
- Claude Code の主要概念（コンテキストウィンドウ、ツール等）を図解付きで学べる
- 初級エンジニアでも理解できる平易な日本語
- 実際の環境の内容を参考例として掲載

## 3. 画面構成

### 3.1 全体レイアウト

```
+----------------------------------------------------------+
|  ヘッダー: Claude Code 設定ファイルガイド                    |
|  [ガイド] [ファイル作成] [概念説明]  ← タブ切り替え           |
+----------------------------------------------------------+
|                                                          |
|  ← タブごとにコンテンツが切り替わる（下記参照）               |
|                                                          |
+----------------------------------------------------------+
```

### 3.2 ガイドタブ

```
+-------------------+--------------------------------------+
|                   |                                      |
|  ディレクトリ       |  サイドパネル                          |
|  ツリー            |                                      |
|                   |  [ファイル名]                          |
|  📁 ~/.claude/    |                                      |
|    📄 CLAUDE.md   |  ● なぜ必要？                          |
|    📁 commands/   |    やさしい説明文                       |
|    📄 settings.json|                                     |
|    📁 projects/   |  ● どう書く？                          |
|    ...            |    書き方の説明 + コード例               |
|                   |                                      |
+-------------------+--------------------------------------+
```

- **左ペイン（30%）**: ディレクトリツリー。折りたたみ可能。グローバル設定・プロジェクトルートの2セクション
- **右ペイン（70%）**: 説明パネル。選択なし時はウェルカムメッセージ
- レスポンシブ: モバイルではツリーとパネルを縦並び

### 3.3 ファイル作成タブ

```
【初期画面: ファイルタイプ選択】
+----------------------------------------------------------+
|  ┌──────────┐  ┌──────────┐  ┌──────────┐               |
|  │ CLAUDE.md │  │ settings │  │  hooks   │               |
|  └──────────┘  └──────────┘  └──────────┘               |
|  ┌──────────┐  ┌──────────┐  ┌──────────┐               |
|  │  rules   │  │ commands │  │  skills  │               |
|  └──────────┘  └──────────┘  └──────────┘               |
+----------------------------------------------------------+

【ウィザード画面: カード選択後】
+----------------------------------------------------------+
|  ← 戻る   CLAUDE.md を作成                                |
|  ● ステップ1 ─── ● ステップ2 ─── ○ ステップ3              |
+---------------------------+------------------------------+
|                           |                              |
|  フォーム入力エリア         |  ガイドパネル                  |
|                           |                              |
|  [テキスト入力]            |  💡 このステップでは…          |
|  [チェックボックス]         |  📝 記入例: ...               |
|  [追加/削除ボタン]          |  ヒント: ...                 |
|                           |                              |
+---------------------------+------------------------------+
|  [← 前へ]   ステップ 1/3   [次へ →]                       |
+----------------------------------------------------------+

【完了画面: 最終ステップ】
+----------------------------------------------------------+
|  生成されたファイル内容                                     |
|  ┌──────────────────────────────────────────┐            |
|  │ # CLAUDE.md                              │            |
|  │ ...                                      │            |
|  └──────────────────────────────────────────┘            |
|  [📋 コピー]  [💾 ダウンロード]                             |
+----------------------------------------------------------+
```

- カード選択 → マルチステップウィザード → 生成結果表示（コピー/ダウンロード）
- 左右分割: フォーム（左） + ガイドパネル（右）で初心者が迷わない構成
- 画面幅1024px以上で横並び、以下で縦並び

### 3.4 概念説明タブ

```
+-------------------+--------------------------------------+
|                   |                                      |
|  概念メニュー       |  詳細パネル                            |
|                   |                                      |
|  🧠 コンテキスト    |  [アイコン + タイトル]                  |
|     ウィンドウ     |                                      |
|  🔧 ツール         |  セクション1: ひとことで言うと           |
|  🔌 MCPサーバー    |    説明文                              |
|  🤖 サブエージェント |                                      |
|                   |  セクション2: なぜ大事？                |
|                   |    説明文 + ASCII図                    |
|                   |                                      |
+-------------------+--------------------------------------+
```

- **左サイドバー**: 概念リスト（アイコン + タイトル + サマリー）
- **右コンテンツ**: 未選択時はウェルカムメッセージ、選択時は詳細（セクション + 図解）

## 4. データ構造

### ツリーデータ (`src/data/treeData.ts`)

```typescript
export type TreeNode = {
  id: string;            // ユニークID
  name: string;          // 表示名（例: "CLAUDE.md"）
  type: "file" | "directory";
  children?: TreeNode[];
};
```

2つのツリーをエクスポート:
- `globalTree` — `~/.claude/` 配下のグローバル設定
- `projectTree` — プロジェクトルート配下の設定

### 説明データ (`src/data/explanations.ts`)

```typescript
export type Step = {
  title: string;         // ステップタイトル
  description: string;   // ステップの説明
  code?: string;         // コード例（任意）
};

export type Explanation = {
  id: string;              // TreeNodeのidと対応
  title: string;           // 表示タイトル
  why: string;             // なぜ必要か（平易な説明）
  what: string;            // 何をするファイル/ディレクトリか
  howToWrite?: string;     // 書き方（ファイルの場合）
  steps?: Step[];          // 作り方のステップバイステップ手順
  example?: string;        // 実際の例（コードブロック）
  tips?: string;           // 初心者向けのヒント
};
```

### 概念データ (`src/data/concepts.ts`)

```typescript
export type ConceptSection = {
  heading: string;       // セクション見出し
  content: string;       // 説明文
  diagram?: string;      // ASCII図（任意）
};

export type Concept = {
  id: string;            // ユニークID
  title: string;         // 概念タイトル
  icon: string;          // 絵文字アイコン
  summary: string;       // 一行サマリー
  sections: ConceptSection[];
};
```

4つの概念を定義:
1. コンテキストウィンドウ (`context-window`)
2. ツール (`tools`)
3. MCPサーバー (`mcp`)
4. サブエージェント (`subagent`)

## 5. ツリーに掲載するファイル/ディレクトリ

### グローバル設定 (`~/.claude/`)

```
~/.claude/
├── CLAUDE.md          ← グローバル指示書
├── settings.json      ← 権限・MCP・プラグイン設定
├── hooks.json         ← ツール実行前後のフック
├── rules/             ← ルールファイル
├── commands/          ← カスタムスラッシュコマンド
│   ├── review.md
│   ├── dead-code.md
│   ├── duplicates.md
│   └── quick-check.md
├── skills/            ← スキル（拡張機能）
├── plugins/           ← プラグイン管理
├── projects/          ← プロジェクトごとの記憶・設定
│   └── {project}/
│       ├── CLAUDE.md  ← プロジェクト別指示
│       ├── settings.json
│       └── memory/    ← Claude の記憶
└── sessions/          ← 会話セッション履歴
```

### プロジェクトルート

```
{プロジェクト}/
├── CLAUDE.md          ← プロジェクトルートの指示書
└── .claude/
    ├── settings.json  ← プロジェクト設定
    ├── rules/         ← プロジェクト固有ルール
    └── commands/      ← プロジェクト固有コマンド
```

## 7. 状態管理

すべて `useState` のみで管理（外部ライブラリ不使用）。

| State | 管理場所 | 型 | 役割 |
|---|---|---|---|
| `activeTab` | App.tsx | `"guide" \| "generator" \| "concepts"` | タブ切り替え |
| `selectedId` | App.tsx | `string \| null` | ガイドタブで選択中のツリーノードID |
| `selectedType` | GeneratorPage | `FileType \| null` | ファイル作成タブで選択中のファイルタイプ |
| `selectedId` | ConceptsPage | `string \| null` | 概念説明タブで選択中の概念ID |
| `currentStep` | 各ウィザード | `number` | ウィザードの現在ステップ |
| `generated` | 各ウィザード | `string` | 生成されたファイル内容 |

## 8. 主要な振る舞い

### 8.1 タブ切り替え
- ヘッダーの3タブ（ガイド / ファイル作成 / 概念説明）をクリックで切り替え
- デフォルトタブ: ガイド

### 8.2 ガイドタブ — ツリー操作
1. ディレクトリノードをクリック → 展開/折りたたみ + 説明表示
2. ファイルノードをクリック → 説明表示
3. 選択中のノードはハイライト
4. `depth < 2` のディレクトリはデフォルト展開

### 8.3 ガイドタブ — サイドパネル
1. 初期状態: ウェルカムメッセージ（「左のツリーから気になるファイルをクリックしてください」）
2. ノード選択時: セクション構成
   - **なぜ必要？** — そのファイル/ディレクトリが存在する理由
   - **何をするもの？** — 役割の説明
   - **書き方** — 具体的な記述方法（ファイルの場合、条件付き表示）
   - **作り方（ステップバイステップ）** — 手順の段階的表示（条件付き表示）
   - **実際の例** — コードブロックで実例を表示
   - **ヒント** — 初心者向けの補足

### 8.4 ファイル作成タブ — ウィザード
1. 初期画面: 6つのファイルタイプをカード形式で選択
2. カード選択後: マルチステップウィザードが起動
3. 各ステップ: 左にフォーム入力、右にガイドパネル（説明・記入例・ヒント）
4. 最終ステップ: OutputPanel で生成結果を表示（コピー + ダウンロード可能）

#### ウィザード一覧

| ファイルタイプ | ステップ数 | 主な入力項目 |
|---|---|---|
| CLAUDE.md | 5 | 役割、ルール、ツール設定、注意事項 |
| settings.json | 3 | 許可ツール（チェックボックス）、MCPサーバー |
| hooks.json | 2 | フック設定（イベント、マッチャー、コマンド） |
| ルールファイル | 3 | テーマ（ファイル名）、ルール内容（Markdown） |
| カスタムコマンド | 3 | コマンド名、説明、指示内容 |
| スキル | 4 | スキル名、説明、トリガーワード、指示内容 |

### 8.5 概念説明タブ
1. 左メニューで概念を選択（アイコン + タイトル + サマリー表示）
2. 右パネルに詳細を表示（複数セクション + ASCII図解）
3. 未選択時はウェルカムメッセージ
