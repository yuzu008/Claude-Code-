# Steering ドキュメント

## プロジェクト概要
- **名前**: Claude Code 設定ファイルガイド
- **目的**: 社内エンジニア（初級者含む）向けに Claude Code の設定ファイル構成を視覚的に理解・作成できるWebアプリ
- **対象ユーザー**: 社内エンジニア（Claude Code 初心者〜中級者）
- **トーン**: 初級エンジニアでも理解できる平易な日本語。専門用語は避けるか補足する

## 技術方針
- **スタック**: Vite + React 19 + TypeScript + CSS Modules
- **状態管理**: useState のみ（外部ライブラリ不使用）
- **ルーティング**: なし（タブ切り替えで useState）
- **外部UIライブラリ**: 不使用
- **パッケージマネージャ**: npm
- **テスト**: Vitest（未導入、次セッションで追加予定）
- **デプロイ**: 未定（Azure Static Web Apps を想定）

## 設計原則
- **データ駆動**: ツリー構造・説明・概念はすべて TS データファイルで管理。コンポーネントはデータを受け取って表示するだけ
- **コンテンツ追加 = データ追加**: 新しいノードや概念の追加は src/data/ にオブジェクトを追加するだけ
- **設計書先行**: 機能追加時は docs/design.md を更新してから実装する
- **初心者ファースト**: すべての説明は「なぜ必要か」から始め、比喩を活用する
