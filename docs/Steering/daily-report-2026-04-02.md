## 解説・学びトピック一覧

### 技術解説
- **再帰コンポーネント**: TreeNodeItem が自身の子を再帰的にレンダリングする手法
- **CSS Modules**: コンポーネントスコープのスタイリング手法（クラス名がハッシュ化される）
- **データ駆動設計**: ツリー構造と説明をTSデータファイルで管理し、UIはデータを受け取って表示するだけのアーキテクチャ

### 設計判断
- **UIライブラリ不使用**: 依存を最小限に抑えるため React + CSS Modules のみで構築
- **タブ切り替え方式**: React Router を使わず useState で単純に切り替え（SPAとして十分）
- **ウィザードの左右分割**: フォーム（左）+ ガイドパネル（右）で初心者が迷わない構成
- **概念説明の左メニュー構成**: ガイドタブと操作感を統一するためカードグリッドから左右分割に変更

### Tips・注意点
- **Vite HMR の罠**: WSL環境ではHMRが効かないことがあり、開発サーバー再起動 + ハードリロード（Ctrl+Shift+R）が必要な場合がある
- **CSS Modules のクラス名**: ビルド後はハッシュ化されるため、Playwright等でDOM検索する際は属性セレクタではなくテキストやロールで検索する
- **git pull --allow-unrelated-histories**: GitHub上でLICENSE等が自動生成されたリポジトリとローカルのコミット履歴をマージする際に必要

### ツール・機能紹介
- **Playwright MCP**: ブラウザ操作によるスクリーンショット撮影・DOM確認でUI実装を検証
- **Vite のプロジェクト初期化**: `npm create vite@latest` でテンプレートから作成（現ディレクトリが空でない場合は別ディレクトリで作成してコピーする回避策）
