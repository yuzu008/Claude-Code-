# セッション開始時のルール

IMPORTANT: セッション開始時に必ず以下を実行すること。

## 1. Steering ドキュメントを読む
以下のファイルを **この順番で** 読み、プロジェクトの全体像・方針・前回の状態を把握する：

1. `docs/Steering/steering.md` — プロジェクト概要・技術方針・設計原則
2. `docs/Steering/` 内の最新の `handoff-*.md` — 前回セッションの引き継ぎ事項
3. `docs/Steering/` 内の最新の `daily-report-*.md` — 前回の作業内容・学び（必要に応じて）

## 2. 設計書を確認する
- `docs/design.md` — 画面構成・データ構造・コンポーネント設計

## 3. 設計上の問題を先に報告する
読んだ結果、重複実装・不整合・設定漏れ等があれば作業開始前にユーザーに報告する。

## 4. セッション終了時
終了時は `docs/Steering/` に以下を作成・更新する：
- `daily-report-{日付}.md` — 日報（作業内容 + 解説・学びトピック一覧）
- `handoff-{日付}.md` — ハンドオフ（プロジェクト状態 + 次にやるべきこと）
- `steering.md` のセッション履歴テーブルに行を追加
